Головний корінь

<html>

Елемент HTML представляє корінь (елемент верхнього рівня) документа HTML, тому його також називають кореневим елементом. Усі інші елементи мають бути нащадками цього елемента.

Метадані документа

Метадані містять інформацію про сторінку. Це включає інформацію про стилі, сценарії та дані, які допомагають програмному забезпеченню (пошуковим системам, браузерам тощо) використовувати та відтворювати сторінку. Метадані для стилів і скриптів можуть бути визначені на сторінці або посилання на інший файл, який містить інформацію.

<base>

Елемент HTML визначає основну URL-адресу для використання для всіх відносних URL-адрес у документі. У документі може бути лише один елемент.

<head>

Елемент HTML містить машиночитану інформацію (метадані) про документ, наприклад його назву, сценарії та таблиці стилів.

<link>

Елемент HTML визначає зв'язки між поточним документом і зовнішнім ресурсом. Цей елемент найчастіше використовується для посилань на CSS, але також використовується для створення піктограм сайтів (як піктограм стилю «favicon», так і піктограм для головного екрана та програм на мобільних пристроях), серед іншого.

<meta>

Елемент HTML представляє метадані, які не можуть бути представлені іншими метаелементами HTML, такими як base, link, script, style або title.

<style>

Елемент HTML містить інформацію про стиль документа або частини документа. Він містить CSS, який застосовується до вмісту документа, що містить елемент.

<title>

Елемент HTML визначає заголовок документа, який відображається в рядку заголовка браузера або на вкладці сторінки. Він містить лише текст; теги в елементі ігноруються.

Секційний корінь

<body>

Елемент HTML представляє вміст документа HTML. У документі може бути лише один елемент.

Секції вмісту

Елементи розділення вмісту дозволяють організувати вміст документа на логічні частини. Використовуйте елементи розділення, щоб створити широкий контур для вмісту вашої сторінки, включаючи навігацію верхнім і нижнім колонтитулами, а також елементи заголовків для визначення розділів вмісту.

<address>

Елемент HTML вказує на те, що вкладений HTML надає контактну інформацію для особи чи людей або для організації.

<article>

Елемент HTML представляє самодостатню композицію в документі, сторінці, програмі або сайті, яка призначена для незалежного розповсюдження або повторного використання (наприклад, у синдикації). Приклади включають: допис на форумі, статтю в журналі чи газеті, запис у блозі, картку продукту, коментар, надісланий користувачем, інтерактивний віджет чи гаджет або будь-який інший окремий елемент вмісту.

<aside>

Елемент HTML представляє частину документа, вміст якої лише опосередковано пов’язаний з основним вмістом документа. Відступи часто представлені у вигляді бічних панелей або вікон викликів.

<footer>

Елемент HTML представляє нижній колонтитул для свого найближчого предка, що розділяє вміст або кореневий елемент розділення. Зазвичай містить інформацію про автора розділу, дані про авторські права або посилання на відповідні документи.

<header>

Елемент HTML представляє вступний вміст, як правило, групу вступних або навігаційних посібників. Він може містити деякі елементи заголовка, а також логотип, форму пошуку, ім’я автора та інші елементи.

<h1>,<h2>, <h3>,<h4>, <h5>,<h6>

Елементи HTML представляють шість рівнів заголовків розділів. є найвищим рівнем розділу та є найнижчим.

<main>

Елемент HTML представляє основний вміст документа. Область основного вмісту складається з вмісту, який безпосередньо пов’язаний із основною темою документа чи основною функціональністю програми або розширює її.

<nav>

Елемент HTML представляє розділ сторінки, метою якого є надання навігаційних посилань у межах поточного документа або до інших документів. Типовими прикладами навігаційних розділів є меню, зміст і покажчики.

<section>

Елемент HTML представляє загальний окремий розділ документа, який не має більш конкретного семантичного елемента для його представлення. Розділи завжди повинні мати заголовок, за дуже рідкісними винятками.

Текстовий зміст

Використовуйте елементи текстового вмісту HTML, щоб організувати блоки або розділи вмісту, розміщені між відкриваючим <body>і закриваючим </body>тегами. Важливі для доступності та SEO, ці елементи визначають мету або структуру цього вмісту.

<blockquote>

Елемент HTML вказує на те, що вкладений текст є розширеною цитатою. Зазвичай це відображається візуально за допомогою відступу (див. Примітки, як це змінити). URL-адресу джерела цитати можна вказати за допомогою атрибута cite, тоді як текстове представлення джерела можна надати за допомогою елемента.

<dd>

Елемент HTML надає опис, визначення або значення для попереднього терміна (dt) у списку описів (dl).

<div>

Елемент HTML є загальним контейнером для вмісту потоку. Це не впливає на вміст або макет, доки не буде певним чином стилізовано за допомогою CSS (наприклад, до нього буде застосовано напряму стиль або якась модель макета, наприклад Flexbox, застосована до його батьківського елемента).

<dl>

Елемент HTML представляє список опису. Елемент містить список груп термінів (зазначених за допомогою елемента dt) та описів (наданих елементами dd). Зазвичай цей елемент використовується для створення глосарію або відображення метаданих (списку пар ключ-значення).

<dt>

Елемент HTML визначає термін в описі або списку визначень і як такий повинен використовуватися всередині елемента dl. За ним зазвичай йде елемент dd; однак кілька елементів <dt> у рядку вказують на кілька термінів, які визначаються безпосереднім наступним елементом dd.

<figcaption>

Елемент HTML представляє заголовок або легенду, що описує решту вмісту його батьківського елемента figure.

<figure>

лемент HTML представляє самодостатній вміст, потенційно з необов’язковим підписом, який вказується за допомогою елемента figcaption. На рисунок, підпис до нього та його вміст посилаються як на єдине ціле.

<hr>

Елемент HTML представляє тематичну перерву між елементами на рівні абзацу: наприклад, зміна сцени в історії або зсув теми в розділі.

<li>

Елемент HTML використовується для представлення елемента в списку. Він має міститися в батьківському елементі: упорядкованому списку (ol), невпорядкованому списку (ul) або меню (menu). У меню та невпорядкованих списках елементи списків зазвичай відображаються за допомогою маркерів. У впорядкованих списках вони зазвичай відображаються зі зростаючим лічильником ліворуч, наприклад цифрою або літерою.

<menu>

Елемент HTML описується в специфікації HTML як семантична альтернатива ul, але розглядається браузерами (і відображається через дерево доступності) як не відмінний від ul. Він представляє невпорядкований список елементів (які представлені елементами li).

<ol>

Елемент HTML представляє впорядкований список елементів, який зазвичай відображається як нумерований список.

<p>

Елемент HTML представляє абзац. Абзаци зазвичай представлені у візуальних медіа як блоки тексту, відокремлені від суміжних блоків порожніми рядками та/або відступом першого рядка, але абзаци HTML можуть бути будь-якими структурними групами пов’язаного вмісту, наприклад зображень або полів форми.

<pre>

Елемент HTML представляє попередньо відформатований текст, який має бути представлений точно так, як написано у файлі HTML. Текст зазвичай відображається за допомогою непропорційного або моноширинного шрифту. Пробіли всередині цього елемента відображаються як написані.

<ul>

Елемент HTML представляє невпорядкований список елементів, який зазвичай відображається як маркований список.

Семантика вбудованого тексту

Використовуйте семантику вбудованого тексту HTML, щоб визначити значення, структуру або стиль слова, рядка чи будь-якого довільного фрагмента тексту.

<a>

Елемент HTML (або елемент прив’язки) за допомогою свого href атрибута створює гіперпосилання на веб-сторінки, файли, адреси електронної пошти, розташування на тій самій сторінці чи будь-що інше, на що може бути адресована URL-адреса.

<abbr>

Елемент HTML представляє абревіатуру або акронім.

<b>

Елемент HTML використовується для привернення уваги читача до вмісту елемента, якому в іншому випадку не надається особливого значення. Раніше це було відомо як елемент жирного шрифту, і більшість браузерів досі малюють текст жирним шрифтом. Однак не слід використовувати <b> для стилізації тексту; натомість вам слід використовувати font-weight властивість CSS, щоб створити жирний шрифт, або strong елемент, щоб вказати, що текст є особливо важливим.

<bdi>

Елемент HTML наказує двонаправленому алгоритму браузера обробляти текст, який він містить, ізольовано від навколишнього тексту. Це особливо корисно, коли веб-сайт динамічно вставляє певний текст і не знає спрямованості тексту, що вставляється.

<bdo>

Елемент HTML замінює поточну спрямованість тексту, тому текст усередині відтворюється в іншому напрямку.

<br>

Елемент HTML створює розрив рядка в тексті (повернення каретки). Це корисно для написання вірша або звернення, де поділ рядків значний.

<cite>

Елемент HTML використовується для опису посилання на цитовану творчу роботу та повинен містити назву цієї роботи. Посилання може бути у скороченій формі відповідно до контексту, що стосується метаданих цитування.

<code>

Елемент HTML відображає свій вміст, оформлений таким чином, щоб вказати, що текст є коротким фрагментом комп’ютерного коду. За замовчуванням текст вмісту відображається за допомогою типового моноширинного шрифту агента користувача.

<data>

Елемент HTML пов’язує даний фрагмент вмісту з машиночитаним перекладом. Якщо вміст пов’язаний із часом або датою, необхідно використовувати цей time елемент.

<dfn>

Елемент HTML використовується для позначення терміна, який визначається в контексті фрази визначення або речення. Елемент p, пара dt/dd або section елемент, який є найближчим предком, вважаються визначенням терміну.

<em>

Елемент HTML позначає текст, який має наголос. Елемент може бути вкладеним, причому кожен рівень вкладеності вказує на більший ступінь акценту.

<i>

Елемент HTML представляє діапазон тексту, який з певних причин відрізняється від звичайного тексту, наприклад ідіоматичний текст, технічні терміни, таксономічні позначення тощо. Історично вони були представлені курсивом, який є оригінальним джерелом назви цього елемента.

<kbd>

Елемент HTML представляє проміжок вбудованого тексту, що позначає текстовий ввід користувача з клавіатури, голосовий ввід або будь-який інший пристрій для введення тексту. Згідно з угодою, агент користувача за замовчуванням відтворює вміст елемента, використовуючи моноширинний шрифт за замовчуванням, хоча це не передбачено стандартом HTML.

<mark>

Елемент HTML представляє текст, який позначено або виділено для довідкових цілей або нотацій, через релевантність або важливість позначеного уривка в контексті, що його охоплює.

<q>

Елемент HTML вказує на те, що вкладений текст є короткою вбудованою цитатою. Більшість сучасних браузерів реалізують це, беручи текст у лапки. Цей елемент призначений для коротких цитат, які не вимагають розриву абзаців; для довгих цитат використовуйте елемент blockquote.

<rp>

Елемент HTML використовується для надання резервних дужок для браузерів, які не підтримують відображення рубінових анотацій за допомогою елемента ruby. Один елемент має охоплювати кожну з відкриваючих і закриваючих дужок, які обгортають елемент rt, що містить текст анотації.

<rt>

Елемент HTML визначає текстовий компонент ruby ​​анотації ruby, який використовується для надання інформації про вимову, переклад або транслітерацію для типографіки Східної Азії. Елемент завжди повинен міститися всередині елемента ruby.

<ruby>

Елемент HTML представляє невеликі анотації, які відображаються над, під або поруч із основним текстом, зазвичай використовуються для показу вимови східноазіатських символів. Його також можна використовувати для анотування інших типів тексту, але це використання менш поширене.

<s>

Елемент HTML відображає текст із закресленням або лінією. Використовуйте елемент, щоб представляти речі, які більше не актуальні або не точні. Однак це не підходить, коли вказується редагування документа; для цього використовуйте елементи del і ins відповідно.

<samp>

Елемент HTML використовується для включення вбудованого тексту, який представляє зразок (або процитований) результат комп’ютерної програми. Його вміст зазвичай відображається за допомогою типового моноширинного шрифту браузера (наприклад, Courier або Lucida Console).

<small>

лемент HTML представляє побічні коментарі та дрібний шрифт, як-от авторські та юридичні тексти, незалежно від стилю представлення. За замовчуванням він відображає текст у ньому на один розмір шрифту меншим, наприклад від small до x-small.

<span>

Елемент HTML — це загальний вбудований контейнер для фразування вмісту, який за своєю суттю нічого не представляє. Його можна використовувати для групування елементів з метою стилізації (за допомогою атрибутів class або id), або тому, що вони мають спільні значення атрибутів, наприклад lang. Його слід використовувати лише тоді, коли жоден інший семантичний елемент не підходить. <span> дуже схожий на елемент div, який є елементом рівня блоку, тоді як <span> є вбудованим елементом.

<strong>

Елемент HTML вказує на те, що його вміст має велику важливість, серйозність або терміновість. Браузери зазвичай відображають вміст жирним шрифтом.

<sub>

Елемент HTML визначає вбудований текст, який має відображатися як нижній індекс виключно з друкарських причин. Нижні індекси зазвичай відображаються зі зниженою базовою лінією, використовуючи менший текст.

<sup>

Елемент HTML визначає вбудований текст, який має відображатися як верхній індекс виключно з типографських причин. Верхні індекси зазвичай відображаються з піднятою базовою лінією, використовуючи менший текст.

<time>

Елемент HTML представляє певний період часу. Він може містити атрибут datetime для перекладу дат у машиночитаний формат, що забезпечує кращі результати пошукової системи або спеціальні функції, такі як нагадування.

<u>

Елемент HTML представляє проміжок вбудованого тексту, який має бути відображено таким чином, щоб вказати, що він має нетекстову анотацію. За замовчуванням це відображається як просте суцільне підкреслення, але його можна змінити за допомогою CSS.

<var>

Елемент HTML представляє назву змінної в математичному виразі або контексті програмування. Зазвичай він представлений із використанням курсивної версії поточного шрифту, хоча така поведінка залежить від браузера.

<wbr>

Елемент HTML представляє можливість розриву слова — позицію в тексті, де браузер може за бажанням розривати рядок, хоча його правила розриву рядка інакше не створюватимуть розрив у цьому місці.

Зображення та мультимедіа

HTML підтримує різні мультимедійні ресурси, такі як зображення, аудіо та відео.

<area>

Елемент HTML визначає область всередині карти зображення, яка має попередньо визначені області, які можна натиснути. Карта зображення дозволяє асоціювати геометричні області зображення з гіперпосиланням.

<audio>

Елемент HTML використовується для вбудовування звукового вмісту в документи. Він може містити одне або кілька джерел звуку, представлених за допомогою атрибута src або елемента source: браузер вибере найбільш підходящий. Він також може бути місцем призначення для потокових медіа за допомогою MediaStream.

<img>

Елемент HTML вставляє зображення в документ.

<map>

Елемент HTML використовується з елементами area для визначення карти зображення (області посилання, яку можна натиснути).

<track>

Елемент HTML використовується як дочірній елемент мультимедійних елементів audio і video. Це дозволяє вказувати тимчасові текстові доріжки (або дані на основі часу), наприклад, для автоматичної обробки субтитрів. Доріжки відформатовано у форматі WebVTT (.vtt файли) — текстові треки веб-відео.

<video>

Елемент HTML вбудовує в документ медіаплеєр, який підтримує відтворення відео. Ви також можете використовувати для аудіовмісту, але елемент audio може забезпечувати більш відповідну взаємодію з користувачем.

Вбудований вміст

Окрім звичайного мультимедійного вмісту, HTML може містити різноманітний інший вміст, навіть якщо з ним не завжди легко взаємодіяти.

<embed>

Елемент HTML вбудовує зовнішній вміст у вказану точку документа. Цей вміст надається зовнішньою програмою або іншим джерелом інтерактивного вмісту, наприклад плагіном браузера.

<iframe>

Елемент HTML представляє вкладений контекст перегляду, вбудовуючи іншу HTML-сторінку в поточну.

<object>

Елемент HTML представляє зовнішній ресурс, який можна розглядати як зображення, вкладений контекст перегляду або ресурс, який обробляється плагіном.

<picture>

Елемент HTML містить нуль або більше елементів source і один елемент img, який пропонує альтернативні версії зображення для різних сценаріїв дисплея/пристрою.

<portal>

Елемент HTML дозволяє вбудовувати іншу HTML-сторінку в поточну з метою забезпечення більш плавної навігації до нових сторінок.

<source>

Елемент HTML визначає кілька медіаресурсів для picture, елемента audio або елемента video. Це порожній елемент, що означає, що він не має вмісту та не має закриваючого тегу. Він зазвичай використовується, щоб пропонувати той самий медіа-вміст у кількох форматах файлів, щоб забезпечити сумісність із широким діапазоном браузерів, враховуючи різну підтримку форматів файлів зображень і форматів медіафайлів.

SVG і MathML

Ви можете вставляти вміст SVG і MathML безпосередньо в документи HTML за допомогою елементів <svg> і <math>.

<svg>

Елемент svgє контейнером, який визначає нову систему координат і вікно перегляду. Він використовується як крайній елемент документів SVG, але його також можна використовувати для вбудовування фрагмента SVG у документ SVG або HTML.

<math>

Елементом верхнього рівня в MathML є <math>. Кожен дійсний екземпляр MathML має бути загорнутий у <math>теги. Крім того, ви не повинні вкладати другий <math>елемент в інший, але ви можете мати довільну кількість інших дочірніх елементів у ньому.

Сценарії

Для створення динамічного вмісту та веб-додатків HTML підтримує використання мов сценаріїв, особливо JavaScript. Певні елементи підтримують цю можливість.

<canvas>

Використовуйте елемент HTML <canvas> або з Canvas API, або з API WebGL, щоб малювати графіку та анімацію.

<noscript>

Елемент HTML визначає розділ HTML, який буде вставлено, якщо тип сценарію на сторінці не підтримується або якщо сценарії наразі вимкнено у веб-переглядачі.

<script>

Елемент HTML використовується для вбудовування виконуваного коду або даних; це зазвичай використовується для вбудовування або посилання на код JavaScript. Елемент також можна використовувати з іншими мовами, такими як мова програмування шейдерів GLSL WebGL і JSON.

Розмежувальні правки

Ці елементи дозволяють вказати, що певні частини тексту були змінені.

<del>

Елемент HTML представляє діапазон тексту, який було видалено з документа. Це можна використовувати, наприклад, під час візуалізації «відстежуваних змін» або інформації про відмінності вихідного коду. Елемент можна використовувати з протилежною метою: для позначення тексту, який було додано до документа.

<ins>

Елемент HTML представляє діапазон тексту, який було додано до документа. Ви можете використовувати цей елемент, щоб аналогічно представити діапазон тексту, який було видалено з документа.

Зміст таблиці

Елементи тут використовуються для створення та обробки табличних даних.

<caption>

Елемент HTML визначає заголовок (або назву) таблиці.

<col>

Елемент HTML визначає стовпець у таблиці та використовується для визначення спільної семантики для всіх загальних клітинок. Зазвичай він міститься в елементі colgroup.

<colgroup>

Елемент HTML визначає групу стовпців у таблиці.

<table>

Елемент HTML представляє табличні дані, тобто інформацію, представлену в двовимірній таблиці, що складається з рядків і стовпців клітинок, що містять дані.

<tbody>

Елемент HTML інкапсулює набір рядків таблиці (tr елементів), вказуючи, що вони складають тіло таблиці (table).

<td>

Елемент HTML визначає клітинку таблиці, яка містить дані. Він бере участь у настільній моделі.

<tfoot>

Елемент HTML визначає набір рядків, що підсумовують стовпці таблиці.

<th>

Елемент HTML визначає комірку як заголовок групи комірок таблиці. Точна природа цієї групи визначається атрибутами scope і headers.

<thead>

Елемент HTML визначає набір рядків, що визначають заголовки стовпців таблиці.

<tr>

Елемент HTML визначає ряд комірок у таблиці. Потім клітинки рядка можна створити за допомогою комбінації елементів td (комірка даних) і th (комірка заголовка).

Форми

HTML надає ряд елементів, які можна використовувати разом для створення форм, які користувач може заповнити та відправити на веб-сайт або в програму. Додаткову інформацію про це можна знайти в посібнику з форм HTML.

<button>

Елемент HTML — це інтерактивний елемент, який активується користувачем за допомогою миші, клавіатури, пальця, голосової команди або іншої допоміжної технології. Після активації він виконує програмовану дію, наприклад надсилання форми або відкриття діалогового вікна.

<datalist>

Елемент HTML містить набір елементів option, які представляють допустимі або рекомендовані параметри, доступні для вибору в інших елементах керування.

<fieldset>

Елемент HTML використовується для групування кількох елементів керування, а також міток (label) у веб-формі.

<form>

Елемент HTML представляє розділ документа, що містить інтерактивні елементи керування для надсилання інформації.

<input>

Елемент HTML використовується для створення інтерактивних елементів керування для веб-форм, щоб приймати дані від користувача; доступний широкий вибір типів вхідних даних і віджетів керування залежно від пристрою та агента користувача. Цей елемент є одним із найпотужніших і найскладніших у всьому HTML завдяки великій кількості комбінацій типів введення та атрибутів.

<label>

Елемент HTML представляє заголовок для елемента в інтерфейсі користувача.

<legend>

Елемент HTML представляє заголовок для вмісту свого батька fieldset.

<meter>

Елемент HTML представляє або скалярне значення у відомому діапазоні, або дробове значення.

<optgroup>

Елемент HTML створює групу опцій всередині елемента select.

<option>

Елемент HTML використовується для визначення елемента, що міститься в елементі select, optgroup або datalist. Таким чином, може представляти пункти меню у спливаючих вікнах та інших списках елементів у документі HTML.

<output>

Елемент HTML — це елемент-контейнер, у який сайт або програма можуть вставляти результати обчислення або результат дії користувача.

<progress>

Елемент HTML відображає індикатор прогресу виконання завдання, який зазвичай відображається у вигляді індикатора виконання.

<select>

Елемент HTML представляє елемент керування, який надає меню параметрів.

<textarea>

Елемент HTML представляє багаторядковий елемент редагування звичайного тексту, корисний, коли ви хочете дозволити користувачам вводити значну кількість тексту довільної форми, наприклад, коментар до рецензії або форму зворотного зв’язку.

Інтерактивні елементи

HTML пропонує вибір елементів, які допомагають створювати інтерактивні об’єкти інтерфейсу користувача.

<details>

Елемент HTML створює віджет розкриття, у якому інформація відображається лише тоді, коли віджет переведено у «відкритий» стан. Короткий опис або мітку необхідно надати за допомогою елемента summary.

<dialog>

Елемент HTML представляє діалогове вікно або інший інтерактивний компонент, наприклад сповіщення, яке можна відхилити, інспектор або підвікно.

<summary>

Елемент HTML визначає короткий виклад, підпис або легенду для вікна розкриття елемента. Натискання елемента перемикає стан батьківського елемента відкритий і закритий.

Веб-компоненти

Веб-компоненти — це технологія, пов’язана з HTML, яка, по суті, дає змогу створювати та використовувати спеціальні елементи так, ніби це звичайний HTML. Крім того, ви можете створювати власні версії стандартних елементів HTML.

<slot>

Елемент HTML — частина набору технологій Web Components — це покажчик місця заповнення всередині веб-компонента, який можна заповнити власною розміткою, що дозволяє створювати окремі дерева DOM і представляти їх разом.

<template>

Елемент HTML — це механізм для зберігання HTML, який не повинен відтворюватися відразу після завантаження сторінки, але може бути створений згодом під час виконання за допомогою JavaScript.
